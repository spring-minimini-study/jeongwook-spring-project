## 1. 스프링의 데이터베이스 접근 방식

스프링은 다양한 DB 접근 방식을 지원하며, 개발 상황에 맞춰 선택할 수 있습니다.

1. **외부 라이브러리 연동**: Mybatis, JPA 등 전문 라이브러리를 사용합니다.
2. **스프링 데이터(Spring Data)**: 인터페이스만 정의하면 구현 클래스를 자동으로 생성해주는 강력한 기능입니다. (JPA, MongoDB 등)
3. **스프링 JDBC**: 자바 표준 JDBC를 스프링이 래핑하여 사용성을 높인 기본 기능입니다.

### 🔄 데이터 접근 흐름
어떤 방식을 사용하더라도 내부적으로는 결국 JDBC를 거치게 됩니다.
> **Service** ➡️ **Repository** ➡️ **(추상화 방법)** ➡️ **JDBC** ➡️ **Database**

*참고: 추상화 단계(JPA 등)가 높아질수록 코드는 간결해지지만, 내부 처리가 복잡해져 세밀한 성능 제어가 어려워질 수 있습니다.*

---

## 2. 스프링 JDBC (JdbcTemplate)

JDBC 표준 코드를 직접 사용할 때 발생하는 중복 작업(연결, 실행, 닫기, 예외 처리)을 대신 처리해주는 도구입니다.

### 🆚 JDBC 직접 사용 vs 스프링 JDBC

- **JDBC 직접 사용**: `Connection`, `PreparedStatement` 객체를 직접 생성하고 닫아야 하며, `try-catch` 블록이 매우 복잡해집니다.
- **스프링 JDBC**: `JdbcTemplate` 하나로 모든 과정을 처리하며, 코드가 매우 직관적입니다.

```java
// 스프링 JDBC를 사용한 간결한 INSERT 예시
public void insert(Training training) {
    jdbcTemplate.update("INSERT INTO training VALUES(?, ?, ?)", 
        training.getId(), 
        training.getTitle(), 
        training.getReserved()
    );
}
```

## 3. 검색 계열 처리 (SELECT)

`JdbcTemplate`은 조회 결과(레코드)를 어떤 형태로 받을지에 따라 다양한 메서드를 제공합니다.

### ① 하나의 컬럼만 가져오기
단일 레코드의 특정 필드값이나, 여러 레코드의 특정 필드 리스트를 조회할 때 사용합니다.

```java
// 1. 단건 조회 (queryForObject)
// t01 아이디를 가진 교육의 타이틀(String)만 가져옴
String title = jdbcTemplate.queryForObject(
    "SELECT title FROM training WHERE id = ?", 
    String.class, 
    "t01"
);

// 2. 다건 조회 (queryForList)
// 모든 교육의 예약 수량(Integer)을 리스트로 가져옴
List<Integer> reservedList = jdbcTemplate.queryForList(
    "SELECT reserved FROM training", 
    Integer.class
);
```

### ② Map 객체로 변환해서 가져오기
별도의 클래스(Entity)를 만들기 번거롭거나, 동적인 결과가 필요할 때 사용합니다. 컬럼명이 Key, 데이터가 Value가 됩니다.

```java
// 1. 단건 레코드를 Map으로 반환 (queryForMap)
Map<String, Object> map = jdbcTemplate.queryForMap(
    "SELECT * FROM training WHERE id = ?", 
    "t01"
);

// 2. 여러 레코드를 Map 리스트로 반환 (queryForList)
List<Map<String, Object>> maps = jdbcTemplate.queryForList(
    "SELECT * FROM training"
);
```

### ③ Entity 객체로 변환해서 가져오기 (DataClassRowMapper)
DB 레코드를 미리 정의한 자바 클래스(Entity)에 자동으로 매핑합니다. 스네이크 케이스(start_date)와 카멜 케이스(startDate)도 똑똑하게 연결해 줍니다.

```java
// 1. 단건을 Entity로 변환 (queryForObject)
Training training = jdbcTemplate.queryForObject(
    "SELECT * FROM training WHERE id = ?",
    new DataClassRowMapper<>(Training.class),
    "t01"
);

// 2. 여러 건을 Entity 리스트로 변환 (query)
List<Training> trainings = jdbcTemplate.query(
    "SELECT * FROM training",
    new DataClassRowMapper<>(Training.class)
);
```


### 4. 갱신 계열 처리 (INSERT, UPDATE, DELETE)
데이터를 변경하는 모든 작업은 update() 메서드 하나로 통일됩니다. 실행 후 **영향을 받은 레코드의 개수(int)**를 반환합니다.

✅ 주요 기능 및 예시
INSERT: 새로운 레코드를 삽입합니다. (키 중복 시 예외 발생)

UPDATE: 기존 레코드를 수정합니다. (대상 레코드가 없으면 에러가 아닌 0 반환)

DELETE: 레코드를 삭제합니다.
```java
// (1) INSERT 예시
int insertCount = jdbcTemplate.update(
    "INSERT INTO training VALUES(?, ?, ?)", 
    "t03", "Spring 교육", 0
);

// (2) UPDATE 예시
int updateCount = jdbcTemplate.update(
    "UPDATE training SET title = ?, start_date_time = ? WHERE id = ?",
    "Spring 실전 교육", 
    LocalDateTime.now(), 
    "t03"
);

// (3) DELETE 예시
int deleteCount = jdbcTemplate.update(
    "DELETE FROM training WHERE id = ?", 
    "t03"
);
```

## 6. JdbcTemplate의 Bean 정의와 주입 (DI)

`JdbcTemplate`은 여러 Repository에서 공유하여 재사용할 수 있는 '공용 도구'입니다. 따라서 DI 컨테이너에 **Bean으로 등록**하여 관리하는 것이 정석입니다.

### ① JavaConfig에서 @Bean으로 정의하기
`JdbcTemplate`은 스프링 프레임워크가 제공하는 라이브러리 클래스이므로, 소스 코드를 직접 수정할 수 없습니다. 따라서 `@Bean` 메서드를 사용하여 수동으로 등록합니다.

```java
@Configuration
public class DbConfig {

    @Bean
    public JdbcTemplate jdbcTemplate(DataSource dataSource) {
        // DI 컨테이너가 이미 관리 중인 DataSource를 인수로 받아 
        // JdbcTemplate을 생성하고 빈으로 반환합니다.
        return new JdbcTemplate(dataSource);
    }
}
```

### ② Repository에서 생성자 주입받기
등록된 JdbcTemplate 빈을 실제 데이터를 처리할 Repository 클래스로 가져와 사용합니다. 이때 생성자 인젝션을 사용하는 것이 가장 좋습니다.

```java
@Repository
public class JdbcTrainingRepository implements TrainingRepository {

    private final JdbcTemplate jdbcTemplate;

    // 생성자 인수로 JdbcTemplate 객체를 받아 필드에 인젝션합니다.
    // 생성자가 하나이므로 @Autowired는 생략 가능합니다.
    public JdbcTrainingRepository(JdbcTemplate jdbcTemplate) {
        this.jdbcTemplate = jdbcTemplate;
    }

    @Override
    public Training findById(String id) {
        // 주입받은 jdbcTemplate을 사용하여 쿼리 실행
        return jdbcTemplate.queryForObject(
            "SELECT * FROM training WHERE id = ?",
            new DataClassRowMapper<>(Training.class),
            id
        );
    }
}
```
