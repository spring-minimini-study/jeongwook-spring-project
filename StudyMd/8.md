8장 스프링과 데이터베이스 접근

Spring에서 사용하는 대표적인 데이터베이스 접근 방식
1. 다른 라이브러리와 연동 (Mybatis, JPA)
2. 스프링 데이터
 - Repository 클레스 자동 생성 기능 (DI 컨테이너 생성시 메모리에 구상 클래스를 생성, Bean 등록)
 - ex) JPA, MongoDB
3. 스프링 JDBC
 - 스프링 프레임워크에서 제공하는 기능
 - 자바 표준인 JDBC를 좀 더 쉽게 사용할 수 있는 방법

Java와 데이터베이스 접근
어떠한 데이터베이스 접근 방식을 사용하더라도 내부에서는
Service -> Repository -> (방법) -> JDBC -> 데이터베이스
고급기능을 사용할수록 Repository와 JDBC 사이가 무거워져 처리 속도 느려지거나 문제가 발생했을 때 내부 처리가 따라오기 어려워지는 경향이 있음.


스프링 JDBC란?
Java가 표준으로 제공한느 JDBC를 래핑하여 JDBC를 직접 사용할 때 정형화되고 중복된 처리를 대신해줌. (처리 속도는 JDBC 직접 사용할때와 거의 차이가 없음)

Service -> Repository -> 스프링 JDBC -> JDBC -> 데이터베이스   
               |                        |
               | -----------------------|
개발자가 작성하는 Repository 클래스 내에서 스프링 JDBC가 제공하는 API를 사용하여 DB 접근 수행
필요에 따라 Repository클래스에서 JDBC를 직접 사용하기도함

Connection, preparedStatmet객체 이용해서 insert 구현
=> 객체 생성 및 execute제외하고서는 중복되는 코드들, 가독성과 유지관리성 저하.
Connection 객체를 닫지 안않아 문제가 발생하기도 한다.

스프링 JDBC에서는
public void insert(Traning traning)
 jdbcTemplate.update("INSERT INTO training VALUES(?,?,?)),
    traning.getId();
    traning.getTitle(),
    ....

jdbcTemplate이 update 메서드를 호출함.
 -> update 메소드 내에서 Connection, PreparedStatement 객체 획득, SQL실행, 객체 닫기, 예외발생시 오류 원인에 따른 비검사 예외로 반환하여 처리 수행함.


BUT, 스프링 프로젝트에서 스프링 JDBC보다 다른 기술을 채택하는 경우가 더 많을거임.

이 때 스프링 JDBC 장점이 살아나는 3가지 경우
1. 처리속도를 최대한 높이고 싶은 경우
 - 스프링 JDBC는 자바 JDBC 직접 사용할때와 비교해서 처리 속도 거의 차이 없음. 매우 속도가 빠른편임
2. JDBC에 익숙한 개발자가 많은 경우
3. 레거시 애플리케이션을 리팩토링하는경우
 - 직접 JDBC를 사용하는 애플리케이션은 중복이 많을텐데 이를 단순화하기 좋음.


JdbcTemplate 클래스
DataSource dataSource...
JdbcTemplate jdbcTemplate = new JdbcTemplate(datSource);
위와 같이 new로 생성하며 dataSource객체를 인수로 넘겨주면됨. (JdbcTemplate클래스 내부에서 DataSource 객체를 사용해 Connection 객체를 가져온다.)

- 검색 계열 처리
1. 하나의 컬럼을 가져오는 경우
2. 레코르를 Map객체로 변환해 가져오는 경우
3. 레코드를 Entitiy 객체로 변환하여 가져오는경우


1. 하나의 컬럼만 가져오기
하나의 레코드에서 하나의 컬럼을 조회할경우
t01 비즈니스예절교육 2021-08-01 1 10 
queryForObject를 사용한다.
String title = jdbcTemplate.queryForObject("SELECT title FROM traning WHERE taning id = ?", String.class, "t01");
첫번째 인수 : 실행할 SQL 문자열
두번째 인수 : 반환할 객체 타입 여기서는 String
세번째 인수 : ? 기호에 대응하는 파라미터 값 
이후는 가변인수로 ? 기호개수만큼 인수를 지정함.

- 여러 레코드에서 하나의 컬럼을 가져오는 경우
t01 비즈니스예절교육 2021-08-01 1 10
t02 자바 교육 2021-08-01 1 10 
t03 마케팅 교육 2021-08-01 1 10
List<Integer> reserved = jdbcTemplate.queryForList("SELECT reserved FROM traning", Integer.class) 
로 여러 컬럼의 reserved 레코드를 List로 받을 수 있다.

2. 레코드를 Map 객체로 변환해서 가져오기
레코드 단위로 가져올 때 
t01 비즈니스예절교육 2021-08-01 1 10
t02 자바 교육 2021-08-01 1 10
t03 마케팅 교육 2021-08-01 1 10
Map<String, Object> map = jdbcTemplate.queryForMap("SELECT * FROM training WHERE id=?",t01);

ID : t01
TITLE : 비즈니스예절교육
...이렇게 컬럼명과 값을 Map으로 가져올 수 잇다.

만약 여러개의 레코들를 가져온다면 아래처럼 List map으로 가져온다.
List<Map<String,Object>> maps = jdbcTemplate.queryForList("SELECT * FROM traning") 

3. 레코드를 Entity 객체로 변환해서 가져오기
t01 비즈니스예절교육 2021-08-01 1 10
위 레코드를 Training 객체로 변환하는데 Training 클래스는 traning 테이블의 컬럼에 해당하는 필드를 가지고있으므로 값을 해당 필드에 저장함.
이 때 자동으로 연결되도록 컬럼명과 같에 함.(스네이크케이스, 카멜케이스로 달라도 문제없이 연결해줌. start_date_time / startDateTime 달라도 대응됨 )
public class Traning{
   private String id,
   ...
}
여기에 Getter와 Setter메서드를 사용, 생성자에서 this.id = id등으로 넣어주기도 가능.

DataClassRowMapper 클래스와 함께 사용하는 방법
Traning traning = jdbcTemplate.queryForObject( "SELECT * FROM training WHERE id = ?",new DataClassRowMapper<>(Training.class),"t01" )
DataClassRowMapper 객체는 레코드의 값을 생성자에서 지정하 클래스의 객체로 자동 변환해주는 스프링 클래스.

List<Traning> trainings = jdbcTemplate.query( "SELECT * FROM training WHERE id = ?",new DataClassRowMapper<>(Training.class),"t01" )
이렇게 여러개 List 객체로도 가져올 수 있다.


갱신 계열 처리
(1) INSERT  "update 메소드 사용"
jdbcTemplate.update("INSERT INTO training VALUES(?,?,?)", "t03","Spring 교육", 0);
반환 : 삽입된 레코드의 수 (int형) 반환됨
키가 중복되어 삽입 안될경우 예외 발생

(2) UPDATE "update 메소드 사용"
int count = jdbcTemplate.update("UPDATE traning SET =?, start_date_time=?","Spring 교육",start_date_time);
반환 : UPDATE로 변경된 레코드의 개수 ( 만약 업데이트된 레코드가 0개일경우 예외가 아닌 0을 반환함)

(3) DELETE "update 메소드 사용"
int count = jdbcTemplate.update("DELETE FROM training WHERE id =?","t03");
반환 : DELETE문으로 삭제된 레코드 개수.
