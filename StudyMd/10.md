# 📚 Spring Framework 학습 기록 (1부: 10장 - 스프링 부트)

핵심 주제: 스프링 부트를 이용한 개발 생산성 극대화와 자동 설정의 이해

---

## 1. 스프링 부트와 스타터(Starter): "개발용 밀키트"

기존 스프링은 라이브러리 추가와 설정 작업이 복잡했지만,  
스프링 부트는 **Starter**라는 개념으로 이를 해결

---

### 📦 스타터의 특징

- **의존성 묶음**
    - 하나의 스타터로 여러 라이브러리 자동 추가

- **버전 관리**
    - `spring-boot-starter-parent`가 버전을 통합 관리

---

### 대표적인 스타터

- `spring-boot-starter-jdbc`
    - DB 연결 기능 제공

- `spring-boot-starter-web`
    - 웹 서버 + MVC 기능 (Tomcat 포함)

- `spring-boot-starter-test`
    - 테스트 환경 구성

---

### pom.xml 예시
```java
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>3.2.0</version>
</parent>

<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-jdbc</artifactId>
    </dependency>
</dependencies>
```
---

## 2. 자동 설정(Auto-configuration): "스마트 홈 시스템"

개발자가 직접 설정하지 않아도, 스프링 부트가 자동으로 설정을 구성

---

### ⚙️ 작동 메커니즘

- 클래스패스를 기반으로 필요한 설정 자동 적용

예시 흐름:
- H2 라이브러리 존재 확인
- → DataSource 자동 생성
- → JdbcTemplate 자동 등록

---

### 특징

- 사용하지 않는 기능은 자동 비활성화
- 성능과 메모리 효율 확보

---

## 3. 자동 설정의 프로퍼티: "리모컨으로 조절하기"

자동 설정된 값을 `application.properties` 또는 `application.yml`로 제어

---

### 기본 규칙

- 위치: `src/main/resources/application.properties`
- 실행 시 자동 로딩

---

### YAML 형식 예시
```java
spring:
datasource:
url: jdbc:h2:mem:testdb
driver-class-name: org.h2.Driver
sql:
init:
mode: always
```
---

## 4. 내장 AP 서버: "휴대용 스피커"

스프링 부트는 서버를 프로그램 내부에 포함

---

### 장점

- 서버 설치 불필요
- 실행만 하면 서버 자동 시작
- 설정을 코드/프로퍼티에서 관리 가능

---

## 5. SpringApplication.run(): "진짜 공장 가동"

스프링 부트 실행의 핵심 메서드

---

### 🚀 주요 역할

- DI 컨테이너 생성
- application.properties 자동 로드
- 자동 설정 활성화

---

## 6. @SpringBootApplication: "마법의 주문"

3가지 핵심 애너테이션을 합친 것

---

### 구성 요소

- `@Configuration`
    - 설정 클래스 선언

- `@ComponentScan`
    - 컴포넌트 자동 탐색

- `@EnableAutoConfiguration`
    - 자동 설정 활성화

---

### 메인 클래스 예시
```java
@SpringBootApplication
public class TrainingApplication {

    public static void main(String[] args) {

        ApplicationContext context = SpringApplication.run(TrainingApplication.class, args);

        ReservationService service = context.getBean(ReservationService.class);
    }
}
```
