# 📚 Spring Framework 학습 기록 (1부: 7장 - 설정의 정석)

핵심 주제: JavaConfig 설정 방식과 @Bean 메서드를 이용한 정교한 빈 제어

---

## 1. JavaConfig 클래스란?: "중앙 지휘 본부"

### 개념
- XML 대신 자바 코드로 스프링 설정을 관리하는 방식

### 선언
- 클래스 상단에 `@Configuration`을 붙여 설정 클래스임을 명시

### 장점
- 컴파일 시점 오류 확인 가능
- if, for 등 자바 문법 활용 가능
- 매우 유연한 설정 가능

---

## 2. JavaConfig를 DI 컨테이너로 불러오는 3가지 전략

설정 클래스를 컨테이너에 등록하는 방식에 따라 구조가 달라짐

---

### ① 생성자 인수 방식

- 직접 설정 클래스를 지정하여 컨테이너 생성
- 테스트나 소규모 프로젝트에 적합
```java
ApplicationContext context =
        new AnnotationConfigApplicationContext(FooConfig.class, BarConfig.class);
```
---

### ② @Import 방식: "모듈화"

- 여러 설정을 하나로 묶는 방식
```java
@Configuration  
@Import(BarConfig.class)  
public class FooConfig {

}

ApplicationContext context = new AnnotationConfigApplicationContext(FooConfig.class);
```
---

### ③ @ComponentScan 방식: "자동화"

- 지정된 패키지를 스캔하여 설정 클래스 자동 등록
- 실무에서 가장 많이 사용
```java
@Configuration  
@ComponentScan(basePackages = "com.wook.config")  
public class MainConfig { }
```
---

## 3. Bean 정의 방법: "주문제작 vs 기성품"

---

### ✅ 스테레오타입 애너테이션

- 대상: 직접 만든 클래스
- 방식: 클래스 위에 애너테이션 부착

비유: "주문제작 가구"

---

### ✅ @Bean 메서드

- 대상: 외부 라이브러리 클래스
- 방식: 직접 객체 생성 후 등록

비유: "이케아 가구"
```java
@Configuration  
public class AppConfig {

    @Bean  
    public DataSource dataSource() {  
        return new EmbeddedDatabaseBuilder()  
                .setType(EmbeddedDatabaseType.H2)  
                .addScripts("schema.sql", "data.sql")  
                .build();  
    }  
}
```
---

## 4. 빈 정의 방식 비교 요약

| 구분 | 스테레오타입 애너테이션 | @Bean 메서드 |
|------|------------------------|-------------|
| 적용 위치 | 클래스 상단 | 설정 클래스 내부 |
| 대상 | 직접 만든 클래스 | 외부 라이브러리 |
| 등록 방식 | 자동 등록 | 수동 등록 |
| 제어권 | 스프링 | 개발자 |

---

## 5. 의존성 주입의 정석: 생성자 인젝션

### 이유

- 객체 생성 시 모든 의존성이 주입됨
- 안정성 확보 (불변성)
- 실무 표준 방식

---

### 예시
```java
@Service  
public class TrainingServiceImpl implements TrainingService {

    private final TrainingRepository trainingRepository;  

    public TrainingServiceImpl(TrainingRepository trainingRepository) {  
        this.trainingRepository = trainingRepository;  
    }  
}
```
