## 1. 프레임워크 기초 및 아키텍처

### 🛠 Spring vs Spring Boot: "조립 PC vs 완제품 PC"

- **Spring**
  - 부품 하나하나(설정)를 직접 골라야 하는 조립 PC
  - 자유도는 높지만 초보자에게는 어려움

- **Spring Boot**
  - 전원을 켜면 바로 쓸 수 있는 완제품 PC
  - **Auto Configuration** 덕분에 설정 없이 바로 개발 가능

---

### 🏗 계층형 아키텍처 (Layered Architecture): "주방의 분업"

효율적인 관리를 위해 역할을 분리

- **Controller (홀 서빙)**
  - 요청을 받고 응답을 반환

- **Service (주방장)**
  - 비즈니스 로직 수행 (핵심 계층)

- **Repository (창고 관리자)**
  - DB 접근 및 데이터 처리

- **Entity (재료 바구니)**
  - DB 데이터를 담는 객체

---

### 🔄 데이터 흐름

브라우저 → Controller → Service → Repository → DB (Entity 활용)

## 2. DI (Dependency Injection)와 Bean

### 💡 DI (의존성 주입): "배달 서비스"

- **개념**
  - 객체를 직접 생성하지 않고 외부에서 주입받는 방식

- **이유**
  - 직접 생성(new)하면 변경 시 코드 수정 필요
  - DI를 사용하면 구현체 변경 시 코드 수정 없이 유지 가능

---

### 📦 Bean & DI 컨테이너: "마법의 공구함"

- **Bean**
  - 스프링이 관리하는 객체

- **DI 컨테이너**
  - Bean을 생성하고 관리하며 주입해주는 공간

---

### 👍 장점

- **싱글톤(Singleton)**
  - 객체를 하나만 생성하여 재사용 → 메모리 절약

---

## 3. 컴포넌트 스캔과 빈 등록: "스티커 붙이기"

### 🔍 @ComponentScan

- 지정된 패키지를 스캔하여 Bean 자동 등록

---

### 🏷 스테레오타입 애너테이션

| 애너테이션 | 대상 | 비유 |
|------------|------|------|
| @Controller | 컨트롤러 | 주문 받는 서빙 |
| @Service | 서비스 | 요리하는 주방장 |
| @Repository | 리포지토리 | 창고 관리자 |
| @Component | 일반 객체 | 다용도 도구 |

---

### 예시: 구상 클래스에 스티커 붙이기
```java
@Service  
public class TrainingServiceImpl implements TrainingService {  
  public void execute() {  
    System.out.println("비즈니스 로직 수행 중...");  
  }  
}
```

## 4. @Autowired를 이용한 의존성 주입

### 🧲 생성자 주입 (Constructor Injection) - 권장 방식

- **장점**
  - 불변성 보장 (final 사용)
  - 의존성 누락 시 컴파일 에러
  - 실무 표준 방식

---

### 생성자 주입 예시

```java
@Service  
public class TrainingServiceImpl implements TrainingService {

    private final TrainingRepository trainingRepository;  

    public TrainingServiceImpl(TrainingRepository trainingRepository) {  
        this.trainingRepository = trainingRepository;  
    }  
}
```
---

## 5. 빈(Bean) 충돌 해결: "누구를 고를까요?"

인터페이스 구현체가 여러 개일 때 해결 방법

- **@Primary**
  - 기본 선택 대상 지정

- **@Qualifier("이름")**
  - 특정 Bean 선택

- **변수명 매칭**
  - 변수 이름으로 자동 선택

---

### 예시
```java
@Component  
@Primary  
public class GasolineEngine implements Engine { }

@Component  
@Qualifier("special")  
public class DieselEngine implements Engine { }
```

## 6. DI 컨테이너 구동: "공장 가동하기"

순수 스프링에서는 직접 컨테이너를 실행해야 함

---

### DI 컨테이너 직접 구동 예시
```java
ApplicationContext context = new AnnotationConfigApplicationContext(AppConfig.class);

TrainingService service = context.getBean(TrainingService.class);

service.findAll();  
```
