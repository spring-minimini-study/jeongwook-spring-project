# 🛡️ Spring Security 정리 (핵심 개념 & 실전 설정)

스프링 시큐리티는 인증(Authentication)과 인가(Authorization)를 기반으로  
애플리케이션을 보호하는 **필터 기반 보안 프레임워크**

---

## 1. 인증 vs 인가

### 🔐 인증 (Authentication)
- "당신은 누구인가?"
- ID / Password 확인
- 로그인 과정

---

### 🚪 인가 (Authorization)
- "이 기능을 사용할 권한이 있는가?"
- 권한(Role)에 따라 접근 제한

---

## 2. 시큐리티의 3대 인가 전략

### ① 요청 인가 (Request Authorization)
- URL 기준으로 접근 제어
- 예: `/admin/**` → 관리자만

---

### ② 메서드 인가 (Method Authorization)
- 특정 메서드 실행 전에 권한 체크
- 예: `@PreAuthorize("hasRole('ADMIN')")`

---

### ③ 화면 인가 (View Authorization)
- UI에서 권한에 따라 표시 여부 제어
- 버튼 숨김 / 노출 처리

---

## 3. 시큐리티 필터 체인 (Security Filter Chain)

스프링 시큐리티는 여러 필터가 체인 구조로 동작

---

### 🔗 주요 필터

- **인증 필터**
    - UsernamePasswordAuthenticationFilter
    - 로그인 요청 가로채서 인증 수행

---

- **예외 처리 필터**
    - ExceptionTranslationFilter
    - 인증 실패 시 로그인 페이지 이동 또는 403 응답

---

- **인가 필터**
    - AuthorizationFilter
    - 요청에 대한 최종 접근 권한 판단

---

## 4. 핵심 동작 흐름

1. 요청 발생
2. 필터 체인 진입
3. 인증 처리
4. 인가 검사
5. 통과 시 Controller 실행

---

## 5. 실전 설정 (SecurityConfig)

```java
@Configuration  
@EnableWebSecurity  
public class SecurityConfig {

    @Bean  
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {  
        http  

            .authorizeHttpRequests(auth -> auth  
                .requestMatchers("/", "/index.jsp", "/login-form").permitAll()  
                .requestMatchers("/main/regist/**").hasRole("ADMIN")  
                .anyRequest().authenticated()  
            )  

            .formLogin(form -> form  
                .loginPage("/login-form")  
                .loginProcessingUrl("/login")  
                .defaultSuccessUrl("/index.jsp")  
                .permitAll()  
            )  

            .logout(logout -> logout  
                .logoutUrl("/logout")  
                .logoutSuccessUrl("/index.jsp")  
                .invalidateHttpSession(true)  
                .deleteCookies("JSESSIONID", "rememberEmail")  
            );  

        return http.build();  
    }  
}
```
---

## 6. 핵심 포인트 요약

- 인증 → "누구냐"
- 인가 → "권한 있냐"
- 필터 체인 → 요청을 단계별로 검사
- 설정은 JavaConfig 한 곳에서 관리
- URL / 메서드 / 화면 단위로 보안 적용 가능
