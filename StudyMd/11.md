# 📚 Spring Framework 학습 기록 (1부: 11장 - 스프링 MVC와 Thymeleaf)

핵심 주제: 웹 애플리케이션의 구조적 분리와 화면 생성 원리

---

## 1. MVC 패턴: "레스토랑의 분업 시스템"

웹 애플리케이션을 만들 때 역할을 3가지로 분리하는 구조

---

### 🍽 구성 요소

- **Controller (웨이터)**
  - 요청(HTTP)을 받아 처리 흐름을 제어
  - 결과를 View로 전달

- **Model (주방 & 식재료)**
  - 비즈니스 로직 수행 (Service)
  - 데이터 관리 (Repository, Entity)

- **View (플레이팅)**
  - 데이터를 사용자에게 보여주는 역할 (HTML)

---

## 2. 스프링 MVC와 Thymeleaf

### 🌐 스프링 MVC

- Controller 중심 웹 프레임워크
- Controller도 Bean이므로 DI 가능

---

### 🍃 Thymeleaf

- 서버 데이터를 HTML에 바인딩하는 템플릿 엔진
- HTML 구조를 유지하면서 데이터 출력 가능

---

## 3. 간단한 구현 예시

### 💻 SampleController
```java
@Controller
public class SampleController {

    @RequestMapping("/hello")
    public String hello(Model model) {

        model.addAttribute("msg", "Hello World");

        return "hello";
    }
}
```
---

### 💻 hello.html
```html
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <title>Hello</title>
</head>
<body>
    <p th:text="${msg}">여기에 메시지가 출력됩니다.</p>
</body>
</html>
```

## 4. 스프링 MVC 내부 동작: "총지배인 시스템"

### 🛠 핵심 컴포넌트

- **DispatcherServlet**
  - 모든 요청을 받는 중심 관리자

- **HandlerMapping**
  - 요청 URL에 맞는 컨트롤러 탐색

- **HandlerAdapter**
  - 컨트롤러 메서드 실행

- **ViewResolver**
  - 뷰 이름을 실제 파일 경로로 변환

---

### 🏃 실행 흐름

1. 요청 → DispatcherServlet 수신  
2. HandlerMapping → 컨트롤러 탐색  
3. HandlerAdapter → 메서드 실행  
4. Service 호출 → 데이터 처리  
5. Model에 데이터 저장  
6. View 이름 반환  
7. ViewResolver → HTML 파일 찾기  
8. HTML 렌더링  
9. 브라우저 응답  

---

## 5. Service 객체 인젝션 (DI)

컨트롤러는 비즈니스 로직을 직접 처리하지 않고 Service에 위임

---

### 💻 Controller에서 Service 사용
```java
@Controller
public class TrainingController {

    private final TrainingService trainingService;

    public TrainingController(TrainingService trainingService) {
        this.trainingService = trainingService;
    }

    @RequestMapping("/trainings")
    public String list(Model model) {

        model.addAttribute("trainings", trainingService.findAll());

        return "training-list";
    }
}
```

## 6. 핸들러 메서드(Handler Method)와 매핑(@XxxMapping)

---

### 🎯 핸들러 메서드란?

- **정의**
  - 웹 요청(URL)과 연결되어 실제로 실행되는 Controller 내부 메서드

- **역할**
  - "이 주소로 오면 이 코드를 실행한다"는 것을 정의하는 핵심 로직

---

### 🏷 @RequestMapping: "종합 안내판"

- URL 경로와 HTTP 메서드를 함께 지정

- 기본 형태
```java
@RequestMapping(path = "/hello", method = RequestMethod.GET)
```
- **특징**
  - 세부 설정 가능 (유연함)
  - 코드가 길어질 수 있음

---

### 🏷 @GetMapping / @PostMapping: "단축키"

@RequestMapping을 간단하게 만든 애너테이션

---

#### @GetMapping

- 데이터 조회(Read) 용도
- 주소창 접근, 조회 요청에 사용

---

#### @PostMapping

- 데이터 등록/수정(Write) 용도
- 폼 제출, 로그인, 게시글 작성 등

---

### 🏗 경로 조합: "계층 구조 설계"

- **클래스 레벨**
  - 공통 URL prefix 지정

- **메서드 레벨**
  - 기능별 세부 경로 지정

---

### 💻 매핑 활용 예시

```java
@Controller  
@RequestMapping("/trainings")  
public class TrainingController {  

    private final TrainingService trainingService;  

    public TrainingController(TrainingService trainingService) {  
        this.trainingService = trainingService;  
    }  

    @GetMapping("/list")  
    public String list(Model model) {  
        model.addAttribute("trainings", trainingService.findAll());  
        return "training/list";  
    }  

    @PostMapping("/register")  
    public String register(ReservationInput input) {  
        return "redirect:/trainings/list";  
    }  
}  
```

## 8. 요청 파라미터 가져오기: "손님의 요구사항 읽기"

브라우저 주소창에 `?pageNo=1&maxCount=10`처럼 데이터가 전달될 때 이를 받는 방법

---

### ① @RequestParam: "하나씩 콕 집어서 받기"

- **개념**
  - 특정 파라미터를 지정해서 개별 변수로 받는 방식

- **장점**
  - 직관적이고 간단함
  - defaultValue 설정 가능 → 안정성 확보

---

### 💻 예시
```java
@GetMapping("/display-list")  
public String displayList(  
    @RequestParam(name = "pageNo", defaultValue = "1") int pNo,  
    @RequestParam(name = "maxCount", defaultValue = "10") int max  
) {  
    return "list";  
}  
```
---

### ② 커맨드 객체: "바구니로 한꺼번에 받기"

- **개념**
  - 파라미터와 동일한 필드명을 가진 객체로 자동 바인딩

- **장점**
  - 코드 간결
  - 유지보수 용이

---

### 💻 예시
```java
public class PageInput {  
    private int pageNo;  
    private int maxCount;  
}  

@GetMapping("/display-list")  
public String displayList(PageInput pageInput) {  
    return "list";  
}  
```

## 9. Bean Validation: "불량 데이터 걸러내기"

입력값이 올바른지 검증하는 기능

---

### ✅ 주요 애너테이션

- `@NotBlank` → 공백, null 금지 (문자열)
- `@NotNull` → null 금지
- `@Min`, `@Max` → 숫자 범위 제한
- `@Email` → 이메일 형식 검증
- `@Pattern` → 정규식 기반 검증

---

### 🛠 핵심 구조

- `@Validated` → 검증 시작
- `BindingResult` → 결과 저장

---

### 💻 예시
```java
@PostMapping("/register")  
public String register(  
    @Validated ReservationInput input,  
    BindingResult bindingResult,  
    Model model  
) {  

    if (bindingResult.hasErrors()) {  
        return "registrationForm";  
    }  

    service.save(input);  
    return "redirect:/success";  
}  
```

## 10. View에 데이터 전달 (Model)

---

### 🧺 단일 데이터 전달

- 객체 하나를 View로 전달

---

### 🧺 리스트 전달

- 여러 데이터를 묶어서 전달 가능

---

### 💻 전체 예시
```java
@Controller  
@RequestMapping("/training")  
public class TrainingController {  

    private final TrainingService trainingService;  

    public TrainingController(TrainingService trainingService) {  
        this.trainingService = trainingService;  
    }  

    @GetMapping("/details/{id}")  
    public String details(@PathVariable String id, Model model) {  
        Training training = trainingService.findById(id);  
        model.addAttribute("training", training);  
        return "training/trainingDetails";  
    }  

    @GetMapping("/list")  
    public String list(Model model) {  
        List<Training> list = trainingService.findAll();  
        model.addAttribute("trainings", list);  
        return "training/traininglist";  
    }  
}  
```
