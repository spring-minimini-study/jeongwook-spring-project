# 📚 Spring Framework 학습 기록 (1부: 11장 - 스프링 MVC와 Thymeleaf)

핵심 주제: 웹 애플리케이션의 구조적 분리와 화면 생성 원리

---

## 1. MVC 패턴: "레스토랑의 분업 시스템"

웹 애플리케이션을 만들 때 역할을 3가지로 분리하는 구조

---

### 🍽 구성 요소

- **Controller (웨이터)**
  - 요청(HTTP)을 받아 처리 흐름을 제어
  - 결과를 View로 전달

- **Model (주방 & 식재료)**
  - 비즈니스 로직 수행 (Service)
  - 데이터 관리 (Repository, Entity)

- **View (플레이팅)**
  - 데이터를 사용자에게 보여주는 역할 (HTML)

---

## 2. 스프링 MVC와 Thymeleaf

### 🌐 스프링 MVC

- Controller 중심 웹 프레임워크
- Controller도 Bean이므로 DI 가능

---

### 🍃 Thymeleaf

- 서버 데이터를 HTML에 바인딩하는 템플릿 엔진
- HTML 구조를 유지하면서 데이터 출력 가능

---

## 3. 간단한 구현 예시

### 💻 SampleController
```java
@Controller
public class SampleController {

    @RequestMapping("/hello")
    public String hello(Model model) {

        model.addAttribute("msg", "Hello World");

        return "hello";
    }
}
```
---

### 💻 hello.html
```html
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <meta charset="UTF-8">
    <title>Hello</title>
</head>
<body>
    <p th:text="${msg}">여기에 메시지가 출력됩니다.</p>
</body>
</html>
```
---

## 4. 스프링 MVC 내부 동작: "총지배인 시스템"

### 🛠 핵심 컴포넌트

- **DispatcherServlet**
  - 모든 요청을 받는 중심 관리자

- **HandlerMapping**
  - 요청 URL에 맞는 컨트롤러 탐색

- **HandlerAdapter**
  - 컨트롤러 메서드 실행

- **ViewResolver**
  - 뷰 이름을 실제 파일 경로로 변환

---

### 🏃 실행 흐름

1. 요청 → DispatcherServlet 수신  
2. HandlerMapping → 컨트롤러 탐색  
3. HandlerAdapter → 메서드 실행  
4. Service 호출 → 데이터 처리  
5. Model에 데이터 저장  
6. View 이름 반환  
7. ViewResolver → HTML 파일 찾기  
8. HTML 렌더링  
9. 브라우저 응답  

---

## 5. Service 객체 인젝션 (DI)

컨트롤러는 비즈니스 로직을 직접 처리하지 않고 Service에 위임

---

### 💻 Controller에서 Service 사용
```java
@Controller
public class TrainingController {

    private final TrainingService trainingService;

    public TrainingController(TrainingService trainingService) {
        this.trainingService = trainingService;
    }

    @RequestMapping("/trainings")
    public String list(Model model) {

        model.addAttribute("trainings", trainingService.findAll());

        return "training-list";
    }
}
```

## 6. 핸들러 메서드(Handler Method)와 매핑(@XxxMapping)

---

### 🎯 핸들러 메서드란?

- **정의**
  - 웹 요청(URL)과 연결되어 실제로 실행되는 Controller 내부 메서드

- **역할**
  - "이 주소로 오면 이 코드를 실행한다"는 것을 정의하는 핵심 로직

---

### 🏷 @RequestMapping: "종합 안내판"

- URL 경로와 HTTP 메서드를 함께 지정

- 기본 형태
```java
@RequestMapping(path = "/hello", method = RequestMethod.GET)
```
- **특징**
  - 세부 설정 가능 (유연함)
  - 코드가 길어질 수 있음

---

### 🏷 @GetMapping / @PostMapping: "단축키"

@RequestMapping을 간단하게 만든 애너테이션

---

#### @GetMapping

- 데이터 조회(Read) 용도
- 주소창 접근, 조회 요청에 사용

---

#### @PostMapping

- 데이터 등록/수정(Write) 용도
- 폼 제출, 로그인, 게시글 작성 등

---

### 🏗 경로 조합: "계층 구조 설계"

- **클래스 레벨**
  - 공통 URL prefix 지정

- **메서드 레벨**
  - 기능별 세부 경로 지정

---

### 💻 매핑 활용 예시

```java
@Controller  
@RequestMapping("/trainings")  
public class TrainingController {  

    private final TrainingService trainingService;  

    public TrainingController(TrainingService trainingService) {  
        this.trainingService = trainingService;  
    }  

    @GetMapping("/list")  
    public String list(Model model) {  
        model.addAttribute("trainings", trainingService.findAll());  
        return "training/list";  
    }  

    @PostMapping("/register")  
    public String register(ReservationInput input) {  
        return "redirect:/trainings/list";  
    }  
}  
```
