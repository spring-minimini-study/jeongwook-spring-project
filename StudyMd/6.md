# 📚 Spring Framework 학습 기록 (1부: 6장 - 프로파일 활용)

핵심 주제: 프로파일(Profile)을 이용한 환경별 설정 자동 전환

---

## 1. 프로파일(Profile)이란?: "상황별 모드 설정"

### 정의
- DI 컨테이너에 등록될 Bean들을 그룹으로 묶고, 특정 환경에서만 활성화되는 기능

---

### 📱 비유: 스마트폰 모드

- **일반 모드**
  - 모든 기능(Bean) 활성화

- **방해금지 모드**
  - 일부 기능(Bean)은 비활성화
  - 꼭 필요한 기능만 동작

---

### 🔑 핵심 원리

- **활성화된 프로파일**
  - 해당 이름의 Bean만 생성됨

- **비활성화된 프로파일**
  - Bean 자체가 생성되지 않음

- **공통 Bean**
  - 어떤 환경에서도 항상 생성됨

---

## 2. 왜 환경을 나누나요? (Production vs Dev)

환경에 따라 요구사항이 완전히 다르기 때문

---

### 🏭 Production (운영 환경)

- 실제 사용자 데이터 사용
- 보안 중요
- 무거운 DB (Oracle, MySQL 등)

---

### 🧪 Development (개발 환경)

- 로컬에서 실행
- 가벼운 DB (H2, 메모리 DB)
- 빠른 실행 속도

---

### 🔍 Test (테스트 환경)

- 테스트 전용 환경
- 데이터 생성/삭제 반복

---

## 3. @Profile 애너테이션: "이름표 붙이기"

클래스에 프로파일 이름을 붙여서 환경별로 Bean을 선택

---

### 💻 예시: 환경별 구현체 분리

```java 
// 1. 인터페이스 (공통 규격)
public interface TrainingRepository {
void save(Training training);
}

// 2. 운영 환경용
@Repository
@Profile("production")
public class JdbcTrainingRepository implements TrainingRepository {
public void save(Training training) {
// 실제 DB 저장 로직
}
}

// 3. 개발 환경용
@Repository
@Profile("dev")
public class MockTrainingRepository implements TrainingRepository {
public void save(Training training) {
// 메모리 기반 저장
}
}

// 4. 서비스 (공통)
@Service
public class TrainingServiceImpl implements TrainingService {

    private final TrainingRepository repository;

    public TrainingServiceImpl(TrainingRepository repository) {
        this.repository = repository;
    }
}
```
---

## 4. 프로파일 활성화 방법: "스위치 올리기"

설정만으로는 동작하지 않고, 실행 시 프로파일을 지정해야 함

---

### ① JVM 옵션 방식
```java
Dspring.profiles.active=dev
```
---

### ② 환경 변수 방식
```java
export SPRING_PROFILES_ACTIVE=production
```
---

### ③ 설정 파일 방식 (Spring Boot)
```java
spring.profiles.active=dev
```
---

## 5. 프로파일 활용의 장점

### ✅ 실수 방지
- 개발 환경에서 운영 DB 접근 차단

---

### ✅ 코드 일관성
- 환경마다 코드 수정 불필요
- 설정만 변경하면 동작 변경 가능

---

### ✅ 유연한 개발
- 네트워크 없이도 개발 가능
- mock 데이터로 테스트 가능
