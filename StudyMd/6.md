# 📚 Spring Framework 학습 기록 (1부: 6장)

> **핵심 주제**: 프로파일(Profile)을 이용한 환경별 설정 전환

---

## 1. 프로파일(Profile)이란?
- **정의**: DI 컨테이너 내의 Bean들을 그룹화하여, 특정 상황(환경)에서만 활성화되도록 제어하는 기능입니다.
- **핵심 원리**:
    - **특정 프로파일 활성화 시**: 해당 그룹의 Bean + 공통 Bean(그룹 지정 없는 Bean)이 함께 생성됩니다.
    - **비활성화 시**: 해당 그룹의 Bean은 DI 컨테이너에 등록되지 않습니다.



---

## 2. 환경별 설정 그룹화 (일반적인 활용)
보통 하드웨어 환경이나 DB 연결 정보가 달라지는 경우에 사용합니다.

- **Production (운영)**: 실제 고객이 사용하는 DB와 연결되는 설정.
- **Staging (검증)**: 운영 직전 단계에서 테스트하는 설정.
- **Development (개발)**: 개발자의 로컬 PC에서 가볍게 사용하는 설정.

---

## 3. @Profile 애너테이션 사용법
클래스(구상 클래스 또는 JavaConfig) 위에 붙여서 해당 Bean이 속할 그룹 이름을 지정합니다.

### 💻 예시 코드 (TrainingRepository 전환)

```java
// 1. 운영 환경용 리포지토리
@Repository
@Profile("production") // "production" 프로파일이 활성화될 때만 Bean으로 등록됨
public class JdbcTrainingRepository implements TrainingRepository {
    // 실제 운영 DB 접속 로직
}

// 2. 개발/테스트 환경용 리포지토리
@Repository
@Profile("dev") // "dev" 프로파일이 활성화될 때만 Bean으로 등록됨
public class MockTrainingRepository implements TrainingRepository {
    // 가짜 데이터를 반환하는 로직
}

// 3. 공통 서비스 (프로파일 지정 없음)
@Service
public class TrainingServiceImpl implements TrainingService {
    // 프로파일과 상관없이 '항상' Bean으로 관리됨
}
```
## 4. 프로파일 활성화 방법 (시동 걸기)

설정만 해둔다고 프로파일이 돌아가지는 않습니다. 스프링에게 "지금은 이 그룹을 써!"라고 명령을 내려야 합니다. 주로 사용하는 3가지 방법입니다.

---

### 1. JVM 시스템 프로퍼티 사용 (가장 일반적)
애플리케이션을 실행할 때 옵션(`-D`)을 추가하는 방식입니다. 주로 IDE 실행 설정이나 서버 실행 스크립트에서 사용합니다.

```bash
# production 프로파일을 활성화하여 실행
java -D spring.profiles.active=production -jar board-app.jar

### ① JVM 시스템 프로퍼티 사용 (가장 일반적)
애플리케이션을 실행할 때 옵션(`-D`)을 추가하는 방식입니다. 주로 IDE 실행 설정이나 서버 실행 스크립트에서 사용합니다.
```

### 2. OS 환경변수 사용
서버의 환경변수에 미리 설정해둡니다.

```bash
# Linux/macOS
export SPRING_PROFILES_ACTIVE=production

# Windows (PowerShell)
$Env:SPRING_PROFILES_ACTIVE = "production"
```
